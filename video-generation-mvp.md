# 视频生成平台产品需求文档

## 1. 项目概述

本项目是一个面向普通用户的视频生成网页平台。用户通过兑换码充值积分，在平台内选择不同的视频生成模型，输入提示词后生成视频。平台后台可以动态管理模型、积分套餐、兑换码和计费方式。

本版本需要完整跑通「用户充值积分 - 选择模型 - 生成视频 - 扣除积分 - 查看结果」的闭环，同时提供独立的管理员后台，用于管理模型、积分套餐、兑换码、用户、任务和积分流水。

## 2. 一句话定位

一个支持多模型、积分制、兑换码充值的视频生成网页平台。

## 3. 目标用户

### 3.1 主要用户

想要通过 AI 生成视频的普通用户，包括但不限于：

- 内容创作者
- 短视频运营人员
- 自媒体用户
- 电商素材制作人员
- 对 AI 视频生成感兴趣的个人用户

### 3.2 管理用户

平台运营和管理员，负责：

- 配置可用模型
- 配置模型计费方式
- 配置积分套餐
- 管理兑换码
- 查看用户和积分流水
- 处理用户充值或生成异常

## 4. 产品核心目标

本产品需要实现以下目标：

1. 用户可以通过积分生成视频。
2. 平台可以同时接入和管理多个视频生成模型。
3. 管理员可以动态开启或关闭模型。
4. 每个模型可以独立配置按秒计费或按次计费。
5. 用户只能通过兑换码充值积分。
6. 积分套餐由后台配置，用户点击套餐后跳转到外部支付链接。
7. 用户获得兑换码后，回到平台兑换积分。
8. 前台和后台必须分割，普通用户不能看到或访问后台。

### 4.1 产品边界

本产品只做视频生成，不做图片生成、AI 聊天、文本生成、语音生成、API 开放平台或其它非视频工具。

说明：平台不提供独立的语音生成工具，但可以把图片、视频、音频文件作为视频生成模型的输入素材。例如某些上游模型可能支持「文 + 图 + 视频 + 音频文件」共同生成视频，这仍然属于视频生成能力。

用户前台只围绕视频生成展开：

- 视频生成
- 视频生成历史
- 积分充值
- 兑换码兑换
- 账号和个人中心

后台只围绕视频业务管理展开：

- 视频模型管理
- 视频参数配置
- 视频任务管理
- 积分和兑换码管理
- 用户管理

## 5. 产品角色

### 5.1 普通用户

普通用户可以：

- 注册和登录
- 查看积分余额
- 通过兑换码充值积分
- 查看可用的视频模型
- 输入提示词生成视频
- 查看生成记录
- 预览或下载生成成功的视频

### 5.2 管理员

管理员可以：

- 管理视频生成模型
- 开启或关闭模型
- 配置模型按秒或按次计费
- 管理积分套餐
- 配置外链支付地址
- 创建和管理兑换码
- 查看用户列表
- 查看积分流水
- 通过生成兑换码的方式给用户补发或赠送积分

### 5.3 前后台隔离原则

平台需要明确分为「用户前台」和「管理员后台」两个部分。

普通用户只能访问用户前台，不能看到后台入口，也不能通过直接输入后台 URL 的方式进入后台。

管理员后台仅允许管理员账号访问。普通用户账号即使已经登录前台，也不能访问后台页面和后台接口。

前后台隔离要求：

- 用户前台和管理员后台使用不同的页面入口。
- 用户前台不展示任何后台入口、后台菜单或后台链接。
- 普通用户账号不能登录后台。
- 管理员账号登录后台后，才可以访问后台功能。
- 后台页面必须做权限校验，不能只依赖前端隐藏。
- 后台接口必须做管理员权限校验，普通用户不能调用后台接口。
- 用户端接口和后台管理接口需要分开设计。
- 用户只能查看和操作自己的数据。
- 管理员可以在后台查看和管理平台数据。

推荐入口设计：

| 系统 | 使用对象 | 示例入口 | 说明 |
| --- | --- | --- | --- |
| 用户前台 | 普通用户 | `/` | 用户生成视频、充值、兑换积分 |
| 管理员后台 | 管理员 | `/admin` | 管理模型、套餐、兑换码、用户和任务 |

如果条件允许，也可以使用独立域名：

| 系统 | 示例域名 |
| --- | --- |
| 用户前台 | `https://www.example.com` |
| 管理员后台 | `https://admin.example.com` |

## 6. 用户端功能范围

### 6.1 注册与登录

本版本用户端采用邮箱账号体系。

用户登录和注册方式：

- 邮箱 + 验证码注册。
- 邮箱 + 验证码登录。
- 邮箱 + 密码登录。
- 首次使用邮箱验证码登录时，如果该邮箱不存在，系统自动创建用户账号。

规则：

- 邮箱是普通用户的唯一账号标识。
- 邮箱验证码既可以用于注册，也可以用于登录。
- 用户第一次使用邮箱验证码登录时，不需要单独走注册流程。
- 用户已经存在时，邮箱验证码登录直接进入账号。
- 用户设置过密码后，可以使用邮箱 + 密码登录。
- 邮箱 + 密码登录失败时，不自动创建账号。
- 邮箱验证码有效期为 10 分钟。
- 同一邮箱 60 秒后才可以重新发送验证码。
- 用户主动设置密码。
- MVP 支持忘记密码和重置密码。
- 管理员账号和普通用户账号分开，用户端邮箱登录不能进入后台。

用户登录后可以查看自己的积分余额、生成记录和充值记录。
未登录用户可以浏览模型和积分套餐，但不能提交生成任务。

### 6.1.1 游客态访问和登录拦截

用户未登录时，也可以进入视频生成工作台，看到登录后可使用的大部分界面和能力。

游客态目标：

- 让用户不用登录也能理解平台有哪些模型和参数。
- 让用户可以看到生成工作台的真实页面结构。
- 降低注册前的理解成本。
- 当用户产生明确操作意图时，再引导登录或注册。

游客态可以查看：

- 顶部导航。
- 视频生成工作台。
- 后台开启且允许展示的视频模型。
- 模型状态和模型标签。
- 输入能力组合。
- 基础提示词和高级提示词表单。
- 图片、视频、音频文件等输入控件的入口。
- 视频比例、清晰度、时长等参数选项。
- 预计消耗积分的计算结果。
- 积分套餐和外链支付入口。

游客态不可以完成：

- 提交生成任务。
- 上传图片、视频、音频文件。
- 兑换兑换码。
- 查看生成历史。
- 查看任务详情。
- 查看积分流水。
- 下载生成结果。

登录拦截规则：

- 游客点击「生成」时，弹出登录 / 注册引导。
- 游客点击上传图片、上传视频或上传音频文件时，弹出登录 / 注册引导。
- 游客点击生成历史、个人中心、积分流水、兑换码兑换时，跳转登录页或弹出登录框。
- 游客可以查看积分套餐，但是否允许跳转外部支付链接由后台配置；默认建议点击支付前先登录。
- 登录成功后，尽量回到用户触发登录前所在的页面和已填写内容。

用户端表现建议：

- 顶部展示登录和注册按钮。
- 游客态不隐藏核心工作台，不做纯营销落地页。
- 生成按钮可以显示为可点击状态，但点击后触发登录拦截。
- 已填写的提示词和参数在登录后尽量保留。

### 6.2 视频生成

用户进入视频生成页面后，可以完成以下操作：

1. 选择视频生成模型或模型供应商。
2. 输入视频提示词。
3. 根据模型能力上传图片、视频或音频文件等输入素材。
4. 选择视频比例。
5. 选择视频清晰度。
6. 选择或输入视频时长。
7. 系统实时展示预计消耗积分。
8. 用户点击生成。
9. 系统校验积分是否充足。
10. 积分充足时创建生成任务。
11. 积分不足时提示用户充值。

生成按钮规则：

- 用户未登录时，生成按钮可以展示为可点击状态，但点击后触发登录 / 注册引导，不直接提交任务。
- 用户已登录但积分不足时，生成按钮可以引导充值。

生成按钮在以下情况需要禁用：

- 未选择可用模型。
- 模型处于关闭、维护中或即将推出状态。
- 必填参数未填写。

### 6.2.1 输入能力组合

用户端不把视频生成能力写死成固定模式，而是根据当前模型的输入能力动态展示表单。

模型可以支持不同输入素材组合，例如：

| 能力组合 | 用户端表现 |
| --- | --- |
| 文生视频 | 只展示提示词输入 |
| 文 + 图生视频 | 展示提示词输入和图片上传 |
| 文 + 视频生视频 | 展示提示词输入和视频上传 |
| 文 + 图 + 视频生视频 | 展示提示词输入、图片上传和视频上传 |
| 文 + 图 + 视频 + 音频文件生视频 | 展示提示词输入、图片上传、视频上传和音频文件上传 |

规则：

- 文本提示词是默认输入能力。
- 图片、视频、音频文件是否展示，由当前模型的后台配置决定。
- 如果某个模型不支持图片输入，则用户端不展示图片上传。
- 如果某个模型不支持视频输入，则用户端不展示视频上传。
- 如果某个模型不支持音频文件输入，则用户端不展示音频上传。
- 用户不能提交当前模型不支持的输入素材。
- 不同输入能力组合可以使用不同价格规则，由后台配置。
- 后台需要为未来更多输入类型预留扩展能力，不把模式枚举写死在前台。

### 6.2.2 基础提示词

基础模式下，用户只需要输入一个完整的视频提示词。

提示词输入框需要支持：

- 多行输入
- 占位提示文案
- 字数上限
- 必填校验
- 提交前去除首尾空格

### 6.2.3 高级提示词

MVP 需要支持 Basic / Advanced 两种提示词方式。新手可以使用基础提示词，进阶用户可以通过高级提示词字段组织视频描述。

用户端提示词区域需要提供：

- 基础模式。
- 高级模式。
- 基础 / 高级切换。
- 高级模式下可以查看结构化 JSON。

高级提示词字段建议包括：

| 字段 | 说明 |
| --- | --- |
| 主要科目 | 视频里的主要人物、物体或场景主体，例如一只雄伟的狮子、两个机器人 |
| 行动 | 主体正在做什么，例如漫步在海滩上、进行对话 |
| 设置 / 环境 | 视频发生在哪里，例如霓虹灯闪烁的未来城市、阳光下的草地 |
| 照明 / 时间 | 例如黄金时刻、情绪化的夜景、戏剧化棚拍灯光 |
| 视觉风格 | 例如电影感、写实、3D 动画、赛博朋克 |
| 镜头 | 例如特写、远景、推镜、航拍、POV |
| 附加详情 | 服装、氛围、运动方式、特殊效果等 |

系统可以把高级字段拼接成最终提示词，并保存原始结构化内容，方便用户后续复用。

高级提示词字段需要支持后台配置。

字段配置能力：

- 字段名称。
- 字段说明。
- 字段占位文案。
- 字段控件类型，例如文本输入、多行文本、下拉选择、可搜索选择。
- 字段选项，例如视觉风格、镜头等预设选项。
- 是否允许用户输入自定义值。
- 是否必填。
- 排序值。
- 是否启用。

规则：

- 有些高级字段是纯文本输入，例如主要科目、行动、设置 / 环境、照明 / 时间、附加详情。
- 有些高级字段是选项型输入，例如视觉风格、镜头。
- 选项型字段需要支持搜索和选择。
- 如果后台允许自定义值，用户可以在选项之外输入新内容。
- 前台只展示后台启用的高级提示词字段。

### 6.2.4 视频比例

用户需要选择视频比例。

视频比例由后台按模型配置，不是全站统一。

参考比例：

| 比例 | 说明 |
| --- | --- |
| 16:9 | 横屏视频 |
| 9:16 | 竖屏短视频 |
| 1:1 | 方形视频 |
| 2:3 | 竖版内容 |
| 3:2 | 横版内容 |

模型级配置示例：

| 模型 | 后台配置比例 | 用户端表现 |
| --- | --- | --- |
| A 模型 | 1:1、9:16 | 用户只能选择 1:1 或 9:16 |
| B 模型 | 16:9 | 用户只能选择 16:9 |
| C 模型 | 16:9、9:16、1:1、2:3、3:2 | 用户可以选择全部已配置比例 |

规则：

- 每个模型都可以配置自己的可选视频比例。
- 用户端只展示当前模型支持的视频比例。
- 用户不能选择或提交后台没有配置的比例。
- 如果模型只配置了一个比例，用户端可以默认选中该比例。
- 切换模型后，比例选项需要同步切换。

### 6.2.5 清晰度

用户需要选择视频清晰度。

参考配置：

| 清晰度 | 说明 |
| --- | --- |
| 480p | 标准清晰度 |
| 720p | 高清 |

后台需要可以配置每个模型支持哪些清晰度，并且可以针对不同清晰度设置不同积分价格或倍率。

### 6.2.6 视频时长

用户需要选择或输入视频时长。

视频时长由后台按模型配置，不是全站统一。

参考配置：

| 模型 | 计费方式 | 用户端时长展示 |
| --- | --- | --- |
| A 模型 | 按秒 | 只展示 6 秒、10 秒 |
| B 模型 | 按次 | 可以不展示时长，或展示后台配置的默认时长 |
| C 模型 | 按秒 | 展示 1 秒、2 秒、3 秒等后台配置的选项 |

规则：

- 每个模型都可以有自己的可选时长列表。
- 用户端只展示当前模型支持的时长选项。
- 按秒计费模型必须配置可选时长，用户选择时长后按秒扣费。
- 按次计费模型可以不让用户选择时长，直接按一次固定价格扣费。
- 如果按次计费模型本身仍然需要传视频时长给第三方接口，后台可以配置默认时长或固定时长选项。
- 用户不能输入或提交后台没有配置的时长。

### 6.2.7 预计消耗积分

视频生成页需要实时展示：

- 当前积分余额
- 本次预计消耗积分
- 扣费依据，例如按秒计费、按次计费、清晰度倍率
- 积分不足时的充值入口

当用户切换模型、模式、比例、清晰度或时长时，预计消耗积分需要同步更新。

### 6.3 模型展示规则

用户端只展示后台状态为「开启」的视频模型。

如果某个模型在后台被关闭：

- 用户端不再展示该模型。
- 用户无法使用该模型创建新任务。
- 已经创建的历史任务不受影响。

模型展示建议参考卡片式结构：

- 模型名称
- 供应商名称
- 模型标签
- 当前状态
- 简短说明
- 支持的输入类型，例如文本、图片、视频、音频文件
- 支持的输入能力组合

模型状态建议包括：

| 状态 | 用户端表现 |
| --- | --- |
| 可用 | 可以选择和生成 |
| 维护中 | 展示维护中标签，不可生成 |
| 即将推出 | 展示即将推出标签，不可生成 |
| 关闭 | 不在用户端展示 |

### 6.4 积分余额

用户端需要在明显位置展示当前积分余额。

建议展示位置：

- 页面顶部导航栏
- 视频生成页提交按钮附近
- 充值页面

### 6.5 充值入口

用户点击充值后，进入积分充值页面。

充值页面展示后台配置的积分套餐，例如：

| 套餐名称 | 积分数量 | 展示价格 | 操作 |
| --- | ---: | --- | --- |
| 基础套餐 | 100 积分 | ¥9.9 | 去支付 |
| 标准套餐 | 500 积分 | ¥39.9 | 去支付 |
| 高级套餐 | 1000 积分 | ¥69.9 | 去支付 |

用户点击「去支付」后，跳转到后台配置的外部支付链接。

平台本版本采用「外链支付 + 兑换码充值」方式。用户支付完成后，会从外部渠道获得兑换码，再回到平台兑换积分。

### 6.6 兑换码充值

用户输入兑换码后，系统需要校验：

- 兑换码是否存在
- 兑换码是否已被使用
- 兑换码是否已禁用
- 兑换码是否已过期

校验通过后：

- 给用户增加对应积分
- 标记兑换码为已使用
- 记录使用用户
- 记录使用时间
- 生成一条积分增加流水

### 6.7 生成记录

用户可以查看自己的视频生成记录。

每条记录建议包含：

- 使用模型
- 提示词
- 视频秒数
- 消耗积分
- 创建时间
- 任务状态
- 视频结果

任务状态包括：

- 生成中
- 生成成功
- 生成失败

生成成功后，用户可以：

- 预览视频
- 下载视频

生成失败后，建议展示失败状态，并根据扣费策略处理退款。

MVP 退款规则：

- 生成失败就退款。
- 不区分上游是否已经产生真实成本。
- 退款金额为本次任务实际扣除积分。
- 退款必须生成积分流水。

### 6.8 视频示例和提示词模板

MVP 不做示例视频和视频示例管理。

原因：

- 第一版重点是生成闭环、积分闭环和后台配置能力。
- 不需要为示例内容额外维护视频素材和展示位。

后续如果需要提示词模板，可以单独增加轻量提示词模板能力，但不作为当前 MVP 必做范围。

### 6.9 用户端导航

用户端导航只保留视频业务需要的入口。

建议包含：

- 视频生成
- 生成历史
- 积分充值
- 联系客服

不建议在本产品里展示：

- AI 聊天
- 图片生成
- API 平台
- 其它非视频工具

## 7. 后台功能范围

### 7.1 模型管理

管理员可以管理平台支持的视频模型。

模型字段建议包括：

| 字段 | 说明 |
| --- | --- |
| 模型名称 | 用户端展示的名称 |
| 供应商名称 | 例如 Google AI、xAI、Meta、Kuaishou |
| 模型标识 | 调用视频生成 API 时使用的 model id |
| 模型描述 | 简短介绍，可选 |
| 模型状态 | 可用 / 维护中 / 即将推出 / 关闭 |
| 计费方式 | 按秒 / 按次 |
| 每秒积分价格 | 按秒计费时使用 |
| 每次积分价格 | 按次计费时使用 |
| 可选时长列表 | 例如 6 秒、10 秒，或 1 秒、2 秒、3 秒 |
| 默认时长 | 按次计费或接口需要时长时使用 |
| 支持输入类型 | text / image / video / audio |
| 输入能力组合 | 例如 `["text"]`、`["text","image"]`、`["text","image","video","audio"]` |
| 输入素材限制 | 各输入类型的格式、大小、数量限制 |
| 可选视频比例 | 例如 16:9、9:16、1:1、2:3、3:2 |
| 支持清晰度 | 例如 480p、720p |
| 上游供应商 | 当前模型实际调用的上游服务 |
| 上游接口地址 | 当前模型调用的接口地址或配置标识 |
| 上游参数映射 | 平台统一参数到上游参数的映射规则 |
| 上游鉴权配置 | API Key、Token 或其它鉴权配置 |
| 排序值 | 控制用户端展示顺序 |

规则：

- 状态为可用的模型才可以生成。
- 状态为维护中或即将推出的模型可以展示，但不可生成。
- 状态为关闭的模型不在用户端展示。
- 一个模型只能使用一种当前计费方式。
- 计费方式可以由管理员在后台切换。
- 每个模型可以配置自己的可用参数，用户端只能选择该模型支持的参数。
- 每个模型可以配置自己的输入能力组合，用户端只展示该模型支持的输入素材。
- 文本、图片、视频、音频文件等输入类型需要按模型分别配置是否支持。
- 每个模型可以配置自己的可选时长列表。
- 每个模型可以配置自己的可选视频比例。
- 按秒计费模型必须配置至少一个可选时长。
- 按次计费模型可以不开放时长选择，但可以配置默认时长给接口使用。
- 每个模型需要绑定一个上游供应商配置。
- 前台提交的参数使用平台统一参数，不直接使用上游原始参数。
- 后端根据模型配置，把平台统一参数转换为对应上游需要的请求参数。

### 7.2 计费配置

后台需要支持两种计费方式。

积分与人民币的基础换算关系：

```text
1 元人民币 = 1 积分
```

说明：

- 1:1 是用户购买积分套餐时的基础换算关系。
- 不同套餐可以在展示价格上做运营配置，但系统内入账积分按套餐配置的积分数量为准。
- 视频生成消耗积分不直接按人民币计算，而是根据后台模型计费规则计算。
- 每个模型的计费方式由后台配置，可以是按秒计费，也可以是按次计费。
- 同一时间一个模型只使用一种当前计费方式。

#### 7.2.1 按秒计费

管理员选择「按秒」后，需要输入每秒积分价格。

按秒计费模型还需要配置可选时长列表。

计算方式：

```text
消耗积分 = 视频秒数 × 每秒积分价格
```

示例：

```text
视频秒数：5 秒
每秒价格：10 积分
消耗积分：50 积分
```

不同模型可以配置不同的可选时长：

```text
A 模型：6 秒、10 秒
C 模型：1 秒、2 秒、3 秒、4 秒、5 秒
```

#### 7.2.2 按次计费

管理员选择「按次」后，需要输入每次积分价格。

按次计费模型不按用户选择的秒数扣费，可以不展示秒数选择。

计算方式：

```text
消耗积分 = 每次固定积分价格
```

示例：

```text
每次价格：80 积分
用户生成一次视频，消耗 80 积分
```

如果模型接口仍然要求传入视频秒数，后台需要为按次计费模型配置默认时长。

#### 7.2.3 参数价格规则

除了基础的按秒或按次计费，后台还需要支持对不同参数配置价格影响。

MVP 需要预留差异化价格配置：

- 不同清晰度可以配置不同价格。
- 不同输入能力组合可以配置不同价格。
- 具体倍率、额外积分或价格数值可以后续在后台配置。

建议支持以下方式：

| 参数 | 配置方式 |
| --- | --- |
| 清晰度 | 可配置倍率或额外积分 |
| 视频时长 | 按秒计费时直接参与计算 |
| 输入能力组合 | 可配置不同输入组合价格，例如文 + 图、文 + 视频、文 + 图 + 视频 + 音频文件 |
| 模型 | 每个模型独立配置基础价格 |

示例：

```text
模型：Grok
计费方式：按秒
每秒价格：10 积分
视频时长：10 秒
清晰度倍率：720p × 1.5
消耗积分：10 × 10 × 1.5 = 150 积分
```

如果暂时不填写复杂倍率，也必须在后台预留配置字段，避免后续不同清晰度、不同输入能力组合无法定价。

#### 7.2.4 模型时长配置示例

| 模型 | 计费方式 | 后台配置 | 用户端表现 | 扣费方式 |
| --- | --- | --- | --- | --- |
| A 模型 | 按秒 | 可选时长：6 秒、10 秒；每秒 10 积分 | 用户只能选择 6 秒或 10 秒 | 6 秒扣 60，10 秒扣 100 |
| B 模型 | 按次 | 每次 80 积分；默认时长 5 秒 | 用户不选择秒数，直接生成 | 每次固定扣 80 |
| C 模型 | 按秒 | 可选时长：1 秒、2 秒、3 秒；每秒 20 积分 | 用户只能选择 1/2/3 秒 | 按选择秒数乘以 20 |

### 7.3 积分套餐管理

管理员可以配置用户端展示的积分套餐。

套餐字段建议包括：

| 字段 | 说明 |

### 7.9 上游供应商适配配置

由于后续可能接入不同上游视频生成服务，每个上游的参数名称、枚举值、鉴权方式和返回结构都可能不同。

系统需要采用「平台统一参数 + 上游适配器」的方式设计。

MVP 阶段不强制确定具体上游供应商，后台和数据结构需要先支持上游占位配置。后续补充具体供应商时，只需要补充对应供应商配置、模型映射和适配器实现。

上游任务状态更新方式需要同时兼容两类供应商：

- 回调型供应商：平台提交任务时传入 `callback_url`，上游完成后回调平台。
- 轮询型供应商：平台保存上游任务 ID，由后台定时任务查询上游状态。

平台内部统一使用一套标准视频生成参数：

| 平台统一参数 | 说明 |
| --- | --- |
| model_id | 平台内部模型 ID |
| prompt | 用户最终提示词 |
| prompt_mode | basic / advanced |
| input_types | 本次提交使用的输入类型，例如 `["text"]`、`["text","image"]` |
| input_assets | 输入素材列表，包含 image / video / audio 等文件 |
| aspect_ratio | 视频比例 |
| resolution | 清晰度 |
| duration_seconds | 视频秒数 |
| seed | 随机种子，可选 |
| callback_url | 回调地址，可选 |

不同上游通过适配器转换参数。

示例：

| 平台参数 | 上游 A 参数 | 上游 B 参数 |
| --- | --- | --- |
| prompt | `prompt` | `text` |
| aspect_ratio | `aspect_ratio` | `ratio` |
| duration_seconds | `duration` | `seconds` |
| resolution | `quality` | `size` |
| input_assets.image | `image_url` | `input_image` |
| input_assets.video | `video_url` | `input_video` |
| input_assets.audio | `audio_url` | `input_audio` |

规则：

- 用户前台只认识平台统一参数。
- 视频任务表保存平台统一参数。
- 上游原始请求参数由适配器生成。
- 每个上游可以有独立的参数映射规则。
- 每个上游可以有独立的鉴权配置。
- 每个上游可以配置状态更新方式：callback / polling / both。
- 每个上游可以有独立的任务状态查询规则。
- 回调型上游必须配置回调验签或来源校验规则。
- 轮询型上游必须配置轮询间隔、最长等待时间和重试次数。
- 每个上游可以有独立的结果解析规则。
- 如果某个上游不支持某个参数，该参数不能在对应模型的用户端表单中展示。
- 如果某个上游需要固定参数，后台可以在模型配置中设置默认值。
- 上游返回失败时，需要转换成平台统一失败状态和失败原因。

### 7.10 上游适配器处理流程

用户提交任务后，系统内部处理流程建议如下：

```text
用户提交平台统一参数
  ↓
校验模型和参数能力
  ↓
计算积分并扣费
  ↓
创建平台视频任务
  ↓
读取模型绑定的上游供应商配置
  ↓
调用对应上游适配器
  ↓
适配器把平台参数转换为上游参数
  ↓
请求上游生成接口
  ↓
保存上游任务 ID 和请求快照
  ↓
轮询或接收回调
  ↓
适配器把上游状态转换为平台状态
  ↓
更新视频任务结果
```

### 7.11 上游参数映射配置

后台需要支持按模型或按供应商配置参数映射。

配置字段建议包括：

| 字段 | 说明 |
| --- | --- |
| 供应商名称 | 例如 Supplier A、Supplier B |
| 适配器标识 | 系统内部适配器 key |
| 上游模型 ID | 上游真实 model id |
| 请求地址 | 上游接口地址 |
| 鉴权方式 | API Key、Bearer Token 等 |
| 参数映射 | 平台参数到上游参数的映射 |
| 枚举值映射 | 例如 `9:16` 转成 `portrait` |
| 默认参数 | 上游必填但前台不展示的参数 |
| 状态映射 | 上游状态到平台状态的映射 |
| 结果字段映射 | 上游返回视频地址、封面、错误信息的位置 |

枚举值映射示例：

```json
{
  "aspect_ratio": {
    "16:9": "landscape",
    "9:16": "portrait",
    "1:1": "square"
  },
  "resolution": {
    "480p": "standard",
    "720p": "high"
  }
}
```

建议原则：

- 简单差异通过后台映射配置解决。
- 复杂差异通过代码里的供应商适配器解决。
- 不建议把所有上游差异都暴露给前台。
- 不建议让前台直接拼接上游请求参数。

## 8. 视频生成任务规则

所有视频生成必须采用异步任务方式。

用户点击生成后，系统不等待上游直接返回最终视频，而是先创建平台任务并返回任务 ID。视频生成结果通过任务状态更新，用户在生成记录中查看进度和结果。

### 8.1 创建任务

用户提交生成时，系统需要：

1. 校验用户是否登录。
2. 校验模型是否存在。
3. 校验模型状态是否为可用。
4. 校验本次输入类型组合是否被该模型支持。
5. 校验提示词是否符合要求。
6. 校验图片、视频、音频文件等输入素材是否符合模型要求。
7. 校验视频比例是否被该模型支持。
8. 校验清晰度是否被该模型支持。
9. 校验视频时长是否被该模型支持。
10. 根据模型计费方式和参数价格规则计算消耗积分。
11. 校验用户积分是否充足。
12. 扣除积分。
13. 创建视频生成任务。
14. 根据模型绑定的上游供应商选择适配器。
15. 将平台统一参数转换为上游请求参数。
16. 异步调用上游视频生成服务。
17. 保存上游任务 ID、请求快照和响应快照。
18. 给用户返回平台任务 ID 和初始任务状态。

规则：

- 所有生成请求都先创建平台任务。
- 前台提交生成后立即进入任务详情或生成记录。
- 前台不阻塞等待最终视频生成完成。
- 系统通过轮询、定时任务或上游回调更新任务状态。
- 上游返回同步结果时，也需要写入平台任务状态，仍按异步任务流程处理。
- 用户刷新页面后仍然可以在生成记录里看到任务状态。

### 8.2 扣费策略

本版本建议采用「提交时先扣费，失败后退款」。

原因：

- 方便防止用户重复提交但余额不足。
- 积分流水清晰。
- 任务创建和扣费逻辑更容易闭环。

规则：

- 任务提交成功后立即扣除积分。
- 任务生成成功后不再额外扣费。
- 任务生成失败后自动退还本次消耗积分。
- 退款需要生成积分流水。

### 8.3 任务状态

任务状态建议包括：

| 状态 | 说明 |
| --- | --- |
| pending | 平台任务已创建，等待提交给上游 |
| submitted | 已提交给上游，等待上游处理 |
| processing | 上游生成中 |
| succeeded | 生成成功 |
| failed | 生成失败 |
| refunding | 退款处理中 |
| refunded | 已退款 |

### 8.4 任务状态更新方式

任务状态可以通过以下方式更新：

| 方式 | 说明 |
| --- | --- |
| 前台轮询 | 用户在生成记录或任务详情页定时查询任务状态 |
| 后台定时轮询 | 服务端定时查询上游任务状态 |
| 上游回调 | 上游生成完成后回调平台接口 |
| 管理员手动处理 | 异常情况下管理员手动标记失败或退款 |

规则：

- 前台只能查询自己的任务状态。
- 后台可以查看全站任务状态。
- 上游回调必须校验签名或来源，不能直接信任。
- 同一个任务状态更新需要幂等，避免重复退款或重复写入结果。
- 任务失败后需要触发退款逻辑。
- 长时间无结果的任务需要支持超时处理。

### 8.5 任务超时处理

系统需要配置任务超时时间。

建议规则：

- 任务超过后台配置的最长等待时间仍未成功，则标记为失败或人工处理。
- 超时失败需要退还用户本次消耗积分。
- 超时处理需要记录失败原因和积分流水。

### 8.6 生成结果存储和有效期

生成成功后，平台需要把上游返回的视频结果全部转存到平台自己的对象存储。

规则：

- 不直接长期依赖上游返回的视频地址。
- 平台需要保存转存后的视频 URL。
- 如果有封面图，也需要转存封面图。
- 生成结果文件有效期为 7 天。
- 文件过期后系统自动删除对象存储中的视频和封面文件。
- 过期删除不删除任务记录，用户仍然可以看到历史任务记录和过期状态。
- 文件过期后，用户不能再预览或下载该视频。
- MVP 不允许用户提前删除生成记录或视频文件。
- 下载链接不做防盗链，也不做额外登录鉴权。

## 10. 核心页面

前台和后台需要分割成两个独立访问区域。

用户前台面向普通用户，负责视频生成、充值、兑换和查看记录。

管理员后台面向平台管理员，负责模型、套餐、兑换码、用户和任务管理。

普通用户不能在前台看到后台入口，也不能通过后台路由访问后台页面。

### 10.1 用户端页面

本版本需要以下页面：

1. 登录 / 注册页
2. 视频生成页
3. 生成记录页
4. 积分充值页
5. 兑换码兑换页
6. 个人中心页

用户端页面不展示：

- 后台登录入口
- 后台菜单
- 模型管理入口
- 兑换码管理入口
- 用户管理入口
- 积分流水管理入口

### 10.2 后台页面

本版本需要以下页面：

1. 后台登录页
2. 模型管理页
3. 上游供应商配置页
4. 视频参数配置页
5. 积分套餐管理页
6. 兑换码管理页
7. 用户管理页
8. 视频任务管理页
9. 积分流水页

后台页面访问规则：

- 未登录管理员账号时，访问后台页面必须跳转到后台登录页。
- 普通用户登录状态不能进入后台。
- 管理员登录状态和普通用户登录状态需要区分。
- 后台页面不能被搜索引擎索引。
- 后台接口必须校验管理员身份。

## 11. 核心用户流程

### 11.1 视频生成流程

```text
用户进入视频生成工作台
  ↓
未登录也可以浏览模型、参数和提示词表单
  ↓
用户选择模型并填写提示词和参数
  ↓
用户点击生成或上传素材
  ↓
如果未登录，弹出登录 / 注册引导
  ↓
用户登录成功后回到工作台并保留已填写内容
  ↓
查看预计消耗积分
  ↓
点击生成
  ↓
系统校验积分
  ↓
扣除积分
  ↓
创建生成任务
  ↓
通过模型绑定的适配器调用上游视频生成服务
  ↓

  ↓
用户进入生成记录或任务详情
  ↓
前台轮询任务状态
  ↓
后台轮询或接收上游回调更新任务
  ↓
用户查看生成结果
```

```text
  ↓

  ↓

  ↓
```

```text
用户端实时使用最新配置
```

## 12. 验收标准

### 12.1 用户端验收

- 用户可以注册并登录。
- 未登录用户可以进入视频生成工作台。
- 未登录用户可以查看可展示模型、参数选项、提示词表单和积分套餐。
- 未登录用户点击生成、上传素材、兑换兑换码、生成历史或个人中心时会触发登录引导。
- 登录成功后尽量回到触发登录前页面，并保留已填写内容。
- 用户可以查看积分余额。
- 用户可以输入兑换码充值积分。
- 已使用的兑换码不能重复使用。
- 用户只能看到后台开启的模型。
- 用户不能使用后台关闭的模型。
- 用户可以选择模型并提交视频生成任务。
- 所有生成请求都会创建异步任务。
- 提交生成后用户可以立即获得平台任务 ID。
- 前台可以展示任务状态：等待中、已提交、生成中、成功、失败、已退款。
- 用户刷新页面后仍能看到未完成任务的最新状态。
- 用户端可以根据模型能力展示文生视频、文 + 图生视频、文 + 视频生视频、文 + 图 + 视频 + 音频文件生视频等输入组合。
- 模型不支持某类输入素材时，用户端不能提交该类素材。
- 用户只能选择当前模型后台配置的视频比例。
- 用户不能提交后台未配置的视频比例。
- 用户可以选择清晰度。
- 用户只能选择当前模型后台配置的时长选项。
- 按次计费模型可以不展示时长选择。
- 用户不能提交后台未配置的时长。
- 用户可以使用基础提示词。
- 用户可以使用高级提示词字段生成视频描述。
- 用户可以看到本次预计消耗积分。
- 系统可以根据模型计费方式计算积分消耗。
- 积分不足时不能提交任务。
- 任务成功后用户可以查看视频结果。
- 任务失败后系统可以退还积分。
- 用户可以查看自己的生成记录。
- 用户前台不展示后台入口。
- 普通用户不能访问后台页面。
- 普通用户不能调用后台管理接口。

### 12.2 后台验收

- 管理员可以新增、编辑、开启、关闭模型。
- 管理员可以切换模型计费方式。
- 管理员可以设置每秒价格和每次价格。
- 管理员可以配置模型供应商名称。
- 管理员可以配置模型状态：可用、维护中、即将推出、关闭。
- 管理员可以配置模型支持的输入类型和输入能力组合。
- 管理员可以按模型配置可选视频比例。

## 14. 当前机制说明

### 14.1 支付机制

本版本不直接接入微信支付、支付宝、Stripe 等支付通道，也不处理支付回调。

平台只负责：

- 展示积分套餐
- 跳转外部支付链接
- 校验兑换码
- 根据兑换码增加积分

外部支付平台负责：

- 收款
- 支付结果确认
- 向用户提供兑换码

### 14.2 兑换码发放机制

用户付款后，兑换码由三方支付平台发放。

平台内不负责：

- 确认支付结果
- 自动发放兑换码
- 处理支付后未收到兑换码的问题

平台只负责：

- 后台创建或导入可兑换的兑换码
- 校验用户输入的兑换码
- 兑换成功后增加积分
- 记录积分流水

### 14.3 积分扣除机制

视频生成失败时必须明确退款规则。

本版本使用：

```text
提交任务先扣积分，生成失败自动退回积分。
```

### 14.4 模型成本控制

不同模型的真实调用成本可能不同，后台必须可以独立配置每个模型的计费方式和价格。

### 14.5 关闭模型的影响

关闭模型后，用户不能再创建新任务，但历史任务仍然需要可以查看。

## 15. 产品定义

## 16. MVP 完整范围梳理

本 MVP 不按上线时间拆分，而按产品闭环和必要能力定义范围。MVP 的目标不是做全量商业化平台，而是做出一个可以真实完成视频生成、积分扣费、兑换码充值和后台管理的最小完整产品。

MVP 主链路定义为：

```text
用户注册 / 登录
  ↓
查看积分余额
  ↓
通过兑换码充值积分
  ↓
选择后台开启的视频模型
  ↓
输入提示词并选择模型支持的参数
  ↓
查看预计消耗积分
  ↓
提交视频生成任务
  ↓
系统创建异步任务并扣除积分
  ↓
系统调用上游视频生成服务
  ↓
用户查看任务状态
  ↓
生成成功后查看 / 下载视频
  ↓
生成失败后自动退款并记录流水
```

MVP 必须同时满足五件事：

- 用户可以完成从充值到生成视频的完整闭环。
- 管理员可以配置模型、套餐、兑换码、用户积分和视频任务。
- 系统可以按模型规则计算积分、扣积分、退积分并记录流水。
- 视频生成任务可以异步创建、查询、成功展示和失败处理。
- 普通用户和管理员后台严格隔离，用户只能访问自己的数据。

- 前台提交后不等待最终视频生成完成。
- 平台任务保存用户参数、计费快照、上游信息和任务状态。
- 系统通过后台轮询或上游回调更新任务状态。
- 用户刷新页面后仍能看到最新任务状态。
- 任务超过配置时间未完成时进入超时处理。

#### 积分和流水

- 积分使用整数。
- 用户积分不过期。
- 创建任务前计算预计消耗积分。
- 积分不足不能创建任务。
- 任务创建成功时先扣积分。
- 扣费必须生成积分流水。
- 任务成功后不再额外扣费。
- 任务失败后退还本次任务消耗积分。
- 退款必须生成积分流水。
- MVP 不支持后台直接手动调整用户积分。
- 管理员补偿积分只能通过生成兑换码完成。
- 所有积分流水需要记录变化后余额。

#### 幂等和一致性

- 兑换码只能成功兑换一次。
- 并发兑换同一兑换码时只能一个请求成功。
- 同一任务只能扣费一次。
- 同一任务只能退款一次。
- 上游回调重复到达时不能重复更新结果或重复退款。
- 任务状态更新需要校验合法状态流转。
- 扣费、任务创建和流水记录需要保证一致性。

#### 权限和隔离

- 用户端和后台使用独立入口。
- 用户端不展示后台入口。
- 普通用户不能登录后台。
- 管理员账号不能作为普通用户账号直接使用。
- 用户只能查看自己的任务、积分和兑换记录。
- 后台接口必须校验管理员权限。

#### 上传和文件

- 图片、视频、音频文件等输入素材只在模型支持时展示。
- 上传文件必须限制格式。
- 上传文件必须限制大小。
- 生成结果需要保存视频 URL。
- 生成结果需要保存封面 URL，若上游不返回则可以为空。
- 用户进入任务记录和任务详情时只能查看自己的任务。
- 视频文件下载链接本身不做防盗链，也不做额外登录鉴权。

#### 内容安全

- MVP 不做平台自建内容安全审核。
- 内容安全先完全依靠上游能力。
- 平台只做提示词长度、上传格式和文件大小等基础参数校验。
- 上传文件需要做格式和大小校验。
- 如果上游因内容安全拒绝或失败，平台按生成失败处理并退款。
- 后台需要能查看上游返回的失败原因。

- 积分商城。
- 多语言和多币种。

| 忘记密码 | MVP 支持忘记密码和重置密码 |

| 支付方式 | 外链支付，不接平台内支付回调 |
| 支付页面配置 | 套餐标题、介绍、价格、积分、三方支付平台、外链地址、按钮文案、状态和排序都由后台配置 |

| 扣费方式 | 提交任务先扣费，失败自动退款 |
| 退款范围 | 只退本次任务扣除的积分 |
| 任务取消 | MVP 不支持用户主动取消任务 |

## 20. MVP 关键风险

### 20.1 范围失控风险

MVP 必须围绕视频生成、积分和兑换码闭环。任何非视频能力、社区能力、复杂营销能力、复杂支付能力都不应进入 MVP。

### 20.2 扣费一致性风险

视频任务涉及扣积分、创建任务、调用上游、写流水、失败退款。这里必须设计成事务或可靠补偿流程，否则容易出现：

- 扣了积分但没有任务。
- 创建了任务但没有扣积分。
- 上游失败但没有退款。
- 回调重复导致重复退款。
- 用户并发提交导致余额扣成负数。

### 20.3 兑换码并发风险

兑换码只能使用一次。兑换接口需要数据库唯一约束、事务和状态检查，不能只依赖前端按钮禁用。

### 20.4 上游不稳定风险

上游视频生成通常耗时较长，并且可能出现任务丢失、超时、返回格式变化、视频 URL 过期等问题。系统需要保存请求和响应快照，并提供后台人工排查入口。

### 20.5 内容合规风险

视频生成涉及内容安全。MVP 不做平台自建内容审核，先完全依靠上游内容安全能力，因此需要接受上游审核规则不可控、失败原因不稳定、不同上游审核尺度不一致等风险。

### 20.6 文件有效期风险

上游返回的视频地址可能过期、下载失败或无法控制访问权限。MVP 已确定生成结果全部转存到平台自己的对象存储，并设置 7 天有效期。

## 21. MVP 验收标准

MVP 验收应重点验证产品闭环是否真实可用，而不是只检查页面是否存在。

### 21.1 主链路验收

1. 新用户可以注册并登录。
2. 用户可以查看当前积分余额。
3. 用户可以兑换一个有效兑换码并增加积分。
4. 已使用兑换码不能再次兑换。
5. 用户可以看到后台开启的视频模型。
6. 用户可以选择模型和模型支持的参数。
7. 系统可以展示预计消耗积分。
8. 积分不足时用户不能提交生成任务。
9. 积分充足时用户可以提交生成任务。
10. 提交任务后系统立即扣除积分并生成流水。
11. 提交任务后用户可以看到任务记录。
12. 上游生成成功后任务变为成功。
13. 成功任务的视频结果会转存到平台对象存储。
14. 成功任务可以预览和下载视频。
15. 生成结果文件 7 天后自动删除。
16. 文件过期后用户不能继续预览或下载视频。
17. 上游生成失败后任务变为失败。
18. 失败任务自动退还本次消耗积分。
19. 退款会生成积分流水。

### 21.2 后台验收

1. 管理员可以独立登录后台。
2. 普通用户不能访问后台页面。
3. 普通用户不能调用后台接口。
4. 管理员可以新增和编辑模型。
5. 管理员可以开启和关闭模型。
6. 关闭模型后用户不能创建新任务。
7. 管理员可以配置模型计费方式和价格。
8. 管理员可以配置模型支持的比例、清晰度和时长。
9. 管理员可以配置积分套餐和外链支付地址。
10. 管理员可以创建、批量生成、导出和禁用兑换码。
11. 兑换码支持批次号、渠道和备注。
12. 批量生成兑换码需要二次确认。
13. 管理员可以查看用户积分和积分流水。
14. 管理员不能直接手动调整用户积分。
15. 管理员可以通过生成兑换码并填写备注的方式补发或赠送积分。
16. 管理员可以查看视频任务和上游排查信息。
17. 后台操作日志保留 90 天。

### 21.3 异常验收

1. 同一个兑换码并发兑换时只能成功一次。
2. 同一个任务不能重复扣费。
3. 同一个任务不能重复退款。
4. 上游回调重复到达不会重复退款。
5. 上游创建任务失败不会造成积分丢失。
6. 任务超时后会进入失败或人工处理状态。
7. 用户不能提交模型不支持的比例、清晰度和时长。
8. 用户不能查看其他用户的任务或积分流水。

## 22. 当前 MVP 定义

当前 MVP 可以定义为：

> 一个支持用户注册登录、兑换码充值、积分计费、视频生成异步任务、失败自动退款、生成记录查看和管理员后台管理的视频生成平台。

MVP 的核心不是功能数量，而是完整闭环：用户能充值、生成、查看结果；管理员能配置、排查、处理异常；系统能保证积分、任务和权限不出错。
